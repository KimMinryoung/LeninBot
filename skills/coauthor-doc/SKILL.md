# 문서 공동 작성 워크플로

사용자와 3단계로 문서를 작성한다: **맥락 수집 → 섹션별 작성 → 검증**.

## 언제 사용하나
- 사용자가 문서/보고서/제안서/스펙/RFC 작성을 요청할 때
- "문서 써줘", "보고서 초안", "기획서 작성", "스펙 정리" 등의 트리거
- 단순 질문이 아니라 **구조화된 결과물**이 필요한 경우

워크플로를 제안하되, 사용자가 거부하면 자유 형식으로 진행.

---

## Stage 1 — 맥락 수집

**목표**: 사용자가 아는 것과 에이전트가 아는 것의 격차를 줄인다.

### Step 1-1: 메타 질문

다음을 물어본다:
1. 문서 유형? (기술 스펙, 의사결정 문서, 제안서, 리서치 보고서 등)
2. 독자는 누구?
3. 독자가 읽고 나서 어떤 행동/이해를 하길 원하나?
4. 따를 템플릿이나 포맷이 있나?
5. 제약사항 (분량, 언어, 마감 등)

### Step 1-2: 정보 수집

사용자에게 관련 맥락을 자유롭게 쏟아놓으라고 한다:
- 프로젝트/문제 배경
- 관련 논의나 기존 문서
- 기술적 제약, 아키텍처, 의존성
- 이해관계자 우려사항
- 대안이 왜 안 되는지

**정리할 필요 없다고 안내.** 정리는 에이전트가 한다.

필요하면 `web_search`로 추가 맥락을 수집한다.

### Step 1-3: 보충 질문

초기 정보를 받은 뒤 5~10개 번호 매긴 보충 질문을 한다.
사용자는 간략하게 답할 수 있다: "1: 맞음, 3: 아니요, 5: ~참고"

**전환 조건**: 기본이 아닌 엣지케이스와 트레이드오프를 질문할 수 있을 때 Stage 2로.

---

## Stage 2 — 섹션별 작성

**목표**: 섹션마다 브레인스토밍 → 큐레이션 → 초안 → 반복 수정.

### Step 2-0: 구조 잡기

문서 유형에 맞는 3~5개 섹션을 제안한다.
사용자 동의 후 전체 구조를 마크다운 파일로 생성:

```
write_file("research/{적절한_파일명}.md", content=전체_구조_스캐폴드)
```

각 섹션은 `[작성 예정]` 플레이스홀더로 채운다.

### Step 2-1 ~ 2-6: 각 섹션에 대해 반복

미지수가 가장 많은 섹션부터 시작한다. 요약 섹션은 마지막에.

**각 섹션마다:**

1. **보충 질문** — 이 섹션에 포함할 내용에 대해 3~5개 질문
2. **브레인스토밍** — 포함할 수 있는 항목 5~15개를 번호로 제시
   - 사용자가 잊었을 수 있는 맥락, 아직 언급 안 된 관점 포함
3. **큐레이션** — 사용자에게 유지/삭제/병합 요청
   - "1,4,7 유지", "3 삭제 (1과 중복)", "11+12 합치기" 등
4. **빈틈 확인** — 빠진 것 없는지 질문
5. **초안 작성** — `write_file`로 해당 섹션 작성
6. **반복 수정** — 사용자 피드백에 따라 `read_file` → 수정 → `write_file`
   - 전체를 다시 출력하지 말고 변경점만 설명
   - 3회 연속 큰 변경 없으면 "삭제해도 될 내용이 있나?" 확인

### 완성 직전 점검 (80%+ 완료 시)

전체 문서를 다시 읽고 다음을 체크:
- 섹션 간 흐름과 일관성
- 중복이나 모순
- 불필요한 문장 ("슬롭")
- 모든 문장이 가치를 가지는지

---

## Stage 3 — 검증

**목표**: 문서가 독자 관점에서 실제로 작동하는지 확인.

### Step 3-1: 독자 질문 예측

독자가 이 문서를 읽으며 할 법한 질문 5~10개를 생성한다.

### Step 3-2: 자체 검증

각 질문에 대해 문서만 보고 답변을 시도한다:
- 답이 명확한가?
- 모호하거나 가정하는 배경지식이 있는가?
- 내부 모순이 있는가?

### Step 3-3: 수정

문제 발견 시 Stage 2로 돌아가 해당 섹션 수정.

### 완료 조건

모든 예측 질문에 문서만으로 명확히 답할 수 있을 때 완료.

---

## 최종 보고

완료 시 사용자에게:
- 생성된 파일 경로
- 문서 구조 요약 (섹션 목록)
- 독자 검증 결과
- "최종 검토는 직접 해주세요 — 사실, 링크, 기술적 세부사항 확인"

---

## 절대 금지
- 사용자 동의 없이 워크플로 강제
- 브레인스토밍 목록을 파일에 쓰기 (대화에서만)
- 전체 문서를 매번 다시 출력 (변경점만)
- 맥락 격차를 무시하고 넘어가기
- research/ 외부에 파일 생성
